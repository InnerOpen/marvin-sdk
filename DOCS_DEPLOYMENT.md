# Documentation Deployment Guide

The marvin-sdk documentation is built with **MkDocs Material** and automatically deployed to **GitHub Pages**.

## 📚 View Live Documentation

**URL:** https://inneropen.github.io/marvin-sdk/

---

## How It Works

### The Two-Branch Pattern

**`main` / `develop` branches** (source code):
- TypeScript source files
- Markdown documentation (`docs/*.md`)
- Configuration (`mkdocs.yml`)
- Build scripts

**`gh-pages` branch** (built site):
- **ONLY** the static HTML/CSS/JS site
- Automatically generated by `mkdocs gh-deploy`
- Never manually edited
- Served by GitHub Pages

### Deployment Flow

```
┌──────────────────────────────────────────────┐
│ 1. Developer edits docs/index.md             │
│ 2. Commits to develop                        │
│ 3. Pushes to GitHub                          │
│    ↓                                          │
│ 4. GitHub Actions workflow triggers          │
│    ↓                                          │
│ 5. MkDocs builds markdown → HTML             │
│    ↓                                          │
│ 6. Pushes built site to gh-pages branch      │
│    ↓                                          │
│ 7. GitHub Pages serves from gh-pages         │
│    ↓                                          │
│ 8. Live at inneropen.github.io/marvin-sdk/  │
└──────────────────────────────────────────────┘
```

## GitHub Pages Setup (One-Time)

### Step 1: Enable GitHub Pages

1. Go to: https://github.com/InnerOpen/marvin-sdk/settings/pages
2. Under **Source**, select:
   - **Deploy from a branch**
3. Under **Branch**, select:
   - Branch: **`gh-pages`**
   - Folder: **`/ (root)`**
4. Click **Save**

### Step 2: Trigger First Deployment

The workflow will trigger automatically on the next push to docs. Or trigger manually:

1. Go to: https://github.com/InnerOpen/marvin-sdk/actions/workflows/docs.yml
2. Click **Run workflow**
3. Select branch: `develop`
4. Click **Run workflow**

### Step 3: Verify Deployment

Wait 1-2 minutes, then visit: https://inneropen.github.io/marvin-sdk/

## Automatic Deployment

### When Does It Deploy?

The workflow (`.github/workflows/docs.yml`) deploys automatically when:

- Push to `main` or `develop` branch
- **AND** files in these paths changed:
  - `docs/**` (any documentation file)
  - `mkdocs.yml` (configuration)
  - `README.md`
  - `SECURITY.md`
  - `AUTHENTICATION.md`
  - `CONTRIBUTING.md`

### Manual Deployment

Trigger manually from GitHub Actions:
1. Go to: https://github.com/InnerOpen/marvin-sdk/actions/workflows/docs.yml
2. Click **Run workflow**
3. Select branch and click **Run workflow**

## Local Development

### Install Dependencies

```bash
# Install Python dependencies
pip3 install mkdocs-material mkdocs-minify-plugin mkdocs-redirects
```

### Serve Locally

```bash
cd ~/Code/marvin-sdk

# Start local server with live reload
npm run docs:serve

# Opens at: http://localhost:8000
```

### Build Locally

```bash
# Build static site (outputs to site/)
npm run docs:build

# Preview the built site
cd site && python3 -m http.server 8000
```

### Deploy Manually

```bash
# Build and deploy to gh-pages branch
npm run docs:deploy
```

**Note:** Manual deployment is rarely needed since GitHub Actions handles it automatically.

## Adding New Documentation

### 1. Create Markdown File

```bash
# Create new doc
echo "# My New Page" > docs/new-page.md
```

### 2. Update Navigation (Optional)

Edit `mkdocs.yml`:

```yaml
nav:
  - Home: index.md
  - Reference:
      - Security: reference/security.md
      - My New Page: new-page.md  # Add here
```

### 3. Commit and Push

```bash
git add docs/new-page.md mkdocs.yml
git commit -m "docs: Add new page"
git push origin develop
```

**GitHub Actions will automatically deploy the changes!**

## Troubleshooting

### Docs Not Updating

1. **Check workflow status**: https://github.com/InnerOpen/marvin-sdk/actions
2. **Verify gh-pages branch exists**: Should be created automatically on first deploy
3. **Check GitHub Pages settings**: Must be enabled and pointing to `gh-pages` branch

### Build Errors

Check the workflow logs:
1. Go to Actions tab
2. Click the failed workflow
3. Expand the "Deploy documentation" step
4. Look for MkDocs errors (usually invalid YAML or broken links)

### Local Preview Not Working

```bash
# Reinstall dependencies
pip3 install --upgrade mkdocs-material

# Check for syntax errors
mkdocs build --strict

# Serve with verbose output
mkdocs serve --verbose
```

## Best Practices

1. **Preview locally** before pushing (`npm run docs:serve`)
2. **Use relative links** within documentation
3. **Optimize images** (use WebP, compress)
4. **Keep navigation shallow** (max 2-3 levels)
5. **Add code examples** with syntax highlighting
6. **Test on mobile** (Material theme is responsive)
